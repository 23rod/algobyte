Estructuras de Datos

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el software en objetos que interactúan entre sí. En Python, la POO permite modelar conceptos del mundo real mediante el uso de clases y objetos, promoviendo la reutilización y la organización del código.

Clases y Objetos
Clases: Una clase es una plantilla para crear objetos. Define un tipo de datos abstracto que incluye tanto datos (atributos) como funciones (métodos) que operan sobre esos datos. Las clases encapsulan datos y comportamientos relacionados en una sola unidad.

Objetos: Un objeto es una instancia de una clase. Cada objeto tiene su propio conjunto de datos y puede utilizar los métodos definidos en su clase para manipular esos datos.

Encapsulación
La encapsulación es el principio de ocultar los detalles internos de una clase y exponer solo una interfaz pública para interactuar con los objetos. Esto se logra mediante el uso de modificadores de acceso. Aunque Python no tiene modificadores de acceso estrictos, se utilizan convenciones como el guion bajo para indicar que ciertos atributos o métodos deben considerarse privados.

Herencia
La herencia es un mecanismo que permite crear una nueva clase a partir de una clase existente. La clase nueva, denominada clase derivada, hereda los atributos y métodos de la clase base. La herencia permite la reutilización de código y la creación de jerarquías de clases, facilitando la extensión y modificación del comportamiento.

Polimorfismo
El polimorfismo permite que diferentes clases respondan de manera diferente a la misma llamada de método. En Python, el polimorfismo se puede lograr mediante la redefinición de métodos en clases derivadas. Esto permite a los objetos de diferentes clases ser tratados de manera uniforme mediante interfaces comunes.

Abstracción
La abstracción es el proceso de simplificar un problema al modelar solo los aspectos relevantes y ocultar los detalles innecesarios. En Python, esto se puede lograr mediante la definición de clases abstractas utilizando el módulo abc. Las clases abstractas no pueden ser instanciadas directamente y pueden contener métodos abstractos (métodos sin implementación) que deben ser implementados por las clases derivadas.

Constructores y Destructores
Constructores: Son métodos especiales que se llaman automáticamente cuando se crea un objeto de una clase. En Python, el constructor se define con el método especial __init__, que se utiliza para inicializar los atributos del objeto y realizar cualquier configuración necesaria.

Destructores: Son métodos especiales que se llaman automáticamente cuando un objeto es destruido. En Python, el destructor se define con el método especial __del__, que se utiliza para liberar recursos y realizar limpieza antes de que el objeto se elimine.

Sobrecarga de Operadores
La sobrecarga de operadores permite definir o modificar el comportamiento de los operadores (como +, -, *, etc.) para que funcionen con objetos de una clase definida por el usuario. En Python, esto se logra mediante la definición de métodos especiales como __add__, __sub__, etc., en la clase.
